## 🎯 專案背景

**專案名稱：** 好室友® V2.0.0
**目標用戶：** 尋找室友的台灣租屋族（20-35 歲）
**核心功能：**
- 多維度室友配對系統（性別偏好、地點、預算、生活習慣）
- Email 搜尋個人配對推薦
- 顯示高分配對範例

**配色方案：**
- 主色：暖橘色 #E08E6D
- 背景：米白色漸層 #FAF7F2 → #F3EDE3
- 強調色：粉橘色系

## 🔍 測試項目清單

### 1. 視覺設計 (Visual Design)

請評估：
- [ ] **配色協調性**：暖橘色系是否適合室友配對主題？
- [ ] **品牌一致性**：整體視覺是否傳達「溫暖、可靠、專業」的感覺？
- [ ] **對比度**：文字是否清晰易讀？（特別是在手機上）
- [ ] **間距與留白**：元素之間的距離是否舒適？
- [ ] **圖示使用**：Heart、Mail 等圖示是否恰當？

### 2. 使用者體驗 (UX)

請測試：
- [ ] **首次進入體驗**：用戶能否立刻理解頁面功能？
- [ ] **操作流程**：Email 搜尋流程是否直覺？
- [ ] **資訊架構**：配對卡片的資訊排列是否清晰？
- [ ] **互動反饋**：按鈕 hover、focus 狀態是否明確？
- [ ] **錯誤處理**：輸入不存在的 email 會發生什麼？

### 3. 手機版適配 (Mobile Responsiveness)

重點檢查：
- [ ] **觸控目標大小**：按鈕是否夠大？（建議最小 44x44px）
- [ ] **單手操作**：重要功能是否容易用拇指點到？
- [ ] **橫向滾動**：是否需要左右滑動？（應避免）
- [ ] **文字大小**：在小螢幕上是否清晰？
- [ ] **輸入框體驗**：在手機上輸入 email 是否順暢？

### 4. 內容呈現 (Content)

請評估：
- [ ] **配對卡片設計**：是否吸引人且易於掃視？
- [ ] **資訊密度**：是否過於擁擠或過於稀疏？
- [ ] **分數顯示**：配對分數的呈現是否醒目？
- [ ] **用戶資訊**：性別、預算、地點等資訊是否一目了然？
- [ ] **CTA（行動呼籲）**：「立即加入」按鈕是否夠明顯？

### 5. 性能與載入 (Performance)

請注意：
- [ ] **頁面載入速度**：初次載入是否流暢？
- [ ] **動畫效果**：hover 和 transition 是否自然？
- [ ] **響應速度**：切換模式（Email 搜尋）是否即時？

## 🧪 測試流程

請按照以下順序測試：

1. **訪問頁面**
   ```
   http://localhost:3000
   ```

2. **瀏覽首頁**
   - 觀察 Banner 輪播
   - 查看「配對推薦」區域
   - 檢查整體佈局

3. **測試 Email 搜尋**
   - 嘗試輸入任意 email（測試錯誤處理）
   - 如果想測試成功案例，可以輸入測試數據中的 email（見下方）

4. **檢查配對卡片**
   - 查看資訊完整性
   - 測試 hover 效果
   - 檢查分數顯示

5. **切換裝置尺寸**
   - 從手機到桌面來回切換
   - 觀察響應式變化

6. **提供整體評價和建議**
